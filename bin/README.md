# 🚀 팀 프로젝트 Git 협업 가이드 (이클립스 + Git Bash)

> **💡 우리 팀의 규칙:** `main` 브랜치는 건드리지 않는다! 모든 작업은 개인 브랜치에서 하고, 합치는 것(Merge)은 **깃허브 웹사이트**에서 진행한다.
> 

## 1단계: 프로젝트 최초 세팅 (팀원 기준)

*이 단계는 깃허브에 올라온 프로젝트를 내 컴퓨터로 처음 가져오는 과정입니다. (질문자님은 이미 올리셨으니 패스, 팀원분만 진행)*

### [방법 A] Git Bash가 편한 경우

1. 프로젝트를 저장할 폴더에서 Git Bash를 켭니다.
2. 깃허브 백엔드 레포지토리 주소를 클론합니다.Bash
    
    ```
    git clone [깃허브_레포지토리_주소]
    ```
    
3. 이클립스를 켭니다.
4. 상단 메뉴 `File` -> `Import...`를 클릭합니다.
5. **`Maven` -> `Existing Maven Projects`** (또는 Gradle 프로젝트라면 Gradle) 선택 후 `Next`.
6. `Root Directory`에 방금 클론받은 폴더를 선택하고 `Finish`를 누르면 끝!

### [방법 B] 이클립스 UI로 한 번에 끝내고 싶은 경우

1. 이클립스 상단 메뉴 `File` -> `Import...`를 클릭합니다.
2. **`Git` -> `Projects from Git`** 선택 후 `Next`.
3. **`Clone URI`** 선택 후 `Next`.
4. `URI` 칸에 깃허브 백엔드 주소를 붙여넣기 하면 (ID/비밀번호 입력 후) `Next`.
5. 브랜치 선택 창에서 `main` 확인 후 `Next` -> 저장할 로컬 경로 확인 후 `Next`.
6. 마법사 창에서 **`Import as general project`** 또는 `Import existing Eclipse projects`를 선택해 가져옵니다. 
→ 만약 메이븐(Maven) 기반 스프링 부트 프로젝트라면, 이클립스 UI로 가져올 때 마지막 마법사 창에서 `Import as clean Maven project` 또는 `Configure -> Convert to Maven Project` 과정을 거쳐야 이클립스가 스프링 부트 프로젝트로 올바르게 인식합니다.

## 2단계: 이클립스 Git 연동 활성화 (공통)

*프로젝트를 불러왔다면, 파일의 변경 상태를 이클립스에서 실시간으로 보기 위해 연동을 활성화합니다.*

1. 이클립스 좌측 `Project Explorer`에서 프로젝트 이름을 **마우스 우클릭**합니다.
2. `Team` -> `Share Project...`를 클릭합니다.
3. Repository 선택 창에서 이미 자동 인식된 내역을 확인하거나 선택한 후 `Finish`를 누릅니다.
4. **결과:** 프로젝트 이름 옆에 `[Layover_Backend main]`처럼 브랜치 이름이 표시되면 성공!

## 3단계: 매일 작업 시작 전 - 최신 코드 가져오기 (공통)

*출근하거나 작업을 시작하기 전, 팀원이 밤새 올린 코드가 있을 수 있으니 항상 최신 `main` 상태를 가져옵니다.*

1. 작업 중인 폴더에서 Git Bash를 켭니다.
2. 원본 `main` 브랜치로 이동합니다.Bash
    
    ```
    git checkout main
    ```
    
3. 깃허브의 최신 코드를 다운받습니다.Bash
    
    ```
    git pull origin main
    ```
    

## 4단계: 내 기능 개발하기 - 브랜치 생성 및 작업 (공통)

*이제 본격적으로 내 코드를 짤 차례입니다. 반드시 새 브랜치를 만들어야 합니다.*

### 1. 새 브랜치 만들기

*예: 내가 로그인 기능을 만든다면 브랜치 이름을 `feat/login`으로 짓습니다.*

Bash

```
git checkout -b feat/로그인이름
# 예시: git checkout -b feat/login
```

> **💡 확인:** 브랜치를 만들면 자동으로 해당 브랜치로 이동합니다. Git Bash 창 끝에 `(feat/login)`으로 바뀌었는지 확인하세요.
> 

### 2. 이클립스에서 코드 수정하기

- 이클립스에서 스프링 부트 코드를 열심히 개발합니다.
- 코드를 수정하면 이클립스 파일 옆에 `>` 모양이 생기며 수정되었다고 친절하게 알려줍니다.

## 5단계: 개발 완료! 깃허브에 올리기 (공통)

*오늘 만든 로그인 기능 개발이 끝났다면, 내 브랜치를 깃허브에 업로드합니다.*

1. Git Bash에서 내가 바꾼 파일들을 장바구니에 담고 커밋 메시지를 적습니다.Bash
    
    ```
    git add .
    git commit -m "feat: 로그인 API 기능 구현 완료"
    ```
    
2. **내가 만든 브랜치 이름 그대로** 깃허브에 푸시합니다. (`main`에 푸시하는 게 아닙니다!)Bash
    
    ```
    git push origin feat/login
    ```
    

## 6단계: 깃허브에서 합치기 (Pull Request & Merge)

*내 컴퓨터 작업은 끝났습니다. 이제 대망의 합치기 단계입니다.*

1. **깃허브 웹사이트(Github.com)** 레포지토리 페이지로 이동합니다.
2. 페이지 상단에 초록색 **`Compare & pull request`** 버튼이 자동으로 떠 있습니다. 클릭합니다.
3. 제목과 내용을 적고 (예: "로그인 기능 구현했습니다. 리뷰 부탁드려요!") 우측 하단의 `Create pull request`를 누릅니다.
4. 팀원과 코드를 확인한 후, 이상이 없다면 아래쪽의 초록색 **`Merge pull request`** -> `Confirm merge`를 누르면 드디어 `main` 브랜치에 내 코드가 합쳐집니다!
5. 깃허브에서 `Confirm merge`를 누르고 나면 버튼 자리에 `Delete branch`라는 보라색 버튼이 뜹니다. 쓸데없는 가지가 쌓이지 않도록 웹에서 삭제해주는 기능입니다.

### 🔄 다시 3단계로 돌아가기

코드가 합쳐졌으니, 다음 기능을 만들 때는 다시 3단계(main으로 이동해서 pull 받기)부터 시작하시면 됩니다! 이 순서만 무한 반복하시면 절대 꼬이지 않는 완벽한 협업을 하실 수 있습니다.
